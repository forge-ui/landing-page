# About Forge Landing Page Kit

Forge Landing Page Kit 是一套面向官网生产的 React 页面与组件素材库。

它把已经整理好的成品页面、区块组件、视觉资产和 Codex 使用说明放在同一个工程里，让后续官网搭建可以直接从成熟页面和组件开始。使用者可以先在 `http://localhost:5173/#kit` 浏览页面和组件，复制编号，再让 Codex 根据业务说明完成组合、改文案、换素材和调整页面节奏。

## 为什么做这个项目

官网项目经常有三个问题：

1. 每次都从零开始搭结构，效率低。
2. 页面需要保持系统化的组件风格和稳定的视觉质量。
3. AI 改页面时容易凭感觉重写，导致设计质量越来越差。

这个项目的目标是把可复用的官网能力沉淀下来：页面有编号，组件有分类，资产在本地，规则写进 Skill。这样 Codex 拿到业务说明后，可以基于已有高质量模块做改造，并延续统一的页面结构和视觉语言。

## 它适合谁

- 需要快速生成官网初版的产品团队。
- 需要长期维护多个业务官网的工程团队。
- 希望 Codex 能稳定复用视觉组件的个人或团队。
- 需要把设计资产转成可维护代码库的前端工程师。

## 它怎么工作

1. 打开 `/#kit` 查看页面和组件索引。
2. 选择一个成品页作为基础，例如 `P001`。
3. 选择需要追加或替换的区块，例如 `S018`、`S041`、`S053`。
4. 给 Codex 产品说明、目标用户、语气、CTA 和素材要求。
5. Codex 基于现有组件完成官网改造，并在浏览器里检查效果。

## 设计方向

这套组件偏向轻量、克制、现代的 SaaS 官网风格：

- 留白充足；
- 字体层级清晰；
- 紫色作为主强调色；
- 卡片和按钮圆角适中；
- 动效以淡入淡出和轻量过渡为主；
- 页面重点放在产品信息、信任感和转化路径上。

## 后续演进

项目会继续补充：

- 更多行业化页面组合；
- 更细的组件分类；
- 更稳定的编号目录；
- 更完整的图片与图标资产；
- 更适合 Codex 读取的官网生成规则。
